## 个人记账系统 (BuZhang Database) - 应用需求介绍

#### 项目背景与概述

- 本系统是一款基于 Web 的个人财务管理应用。旨在帮助用户高效地记录日常收支、管理个人资产账户、设定预算目标，并通过可视化的报表分析财务状况，从而培养良好的消费习惯，实现财富的积累与合理分配。

- 系统采用前后端分离架构，前端注重交互体验，后端保障数据安全与业务逻辑处理。

### 一、用户角色

系统主要面向单一角色的普通用户：

  - **个人用户**：需要进行日常记账、资产管理和报表查看的独立个体。每个用户的数据相互隔离，拥有独立的账户体系。

### 二、功能需求

#### 用户权限与安全模块

- 用户在使用系统前必须**验证身份**，确保财务数据的私密性。

- **用户注册：** 支持新用户通过用户名、邮箱和密码创建账户。系统需对密码进行加密存储。

- **用户登录：** 已注册用户通过账号密码登录系统，获取访问令牌（Token）。

- **注销/退出**：支持用户安全退出当前会话。

#### 收支记账模块
这是系统的**核心高频功能**，用于记录每一笔资金流动。

- 记账功能：

  - 支持记录支出 (Expense) 和 收入 (Income) 交易。

  - **必填信息：** 交易金额、交易类型、关联账户（如微信、银行卡）、关联分类（如餐饮、工资）、交易日期。

  - **选填信息：** 备注说明（用于记录具体消费详情）。
- 交易列表：

  - 以**时间轴或列表**形式展示历史交易记录。
      
  - 支持分页加载，避免数据量过大影响性能。

- **交易编辑/删除：**

  - 允许用户对已记录的错误账单进行修改或彻底删除。

  - 删除或修改交易时，需同步更新关联资产账户的余额。

#### 资产账户管理模块

用于管理用户的资金存储渠道，反映真实的资产状况。

- **账户列表：** 展示所有资产账户及其当前余额。

- **新增账户：** 支持用户添加多种类型的账户（如：现金、储蓄卡、信用卡、支付宝、微信零钱等）。

- **账户编辑/删除：** 支持修改账户名称或移除不再使用的账户。
   - 余额联动：当发生收支记录时，对应账户的余额应自动扣减或增加。

#### 分类管理模块

通过对收支进行分类，帮助用户精细化分析消费结构。

- 支持用户新增个性化的收支类别。

- 支持为分类设置名称和图标/表情符号。

- 分类维护：支持删除或重命名现有分类。

#### 预算管理模块

帮助用户控制消费，防止超支。

- 设定预算：用户可以按月设置总支出预算或特定分类的预算。

- 预算监控：在记账或查看报表时，系统应反馈当前支出是否已接近或超过预算金额。

#### 统计报表模块

通过可视化图表直观展示财务状况。

- 收支概览：展示当月/当年的总收入、总支出及结余。

- 分类统计（饼图）：展示各类支出在总支出中的占比，帮助用户识别主要消费领域。

- 趋势分析（折线图/柱状图）：展示每日或每月的收支变化趋势。

### 三、非功能需求

#### 安全性

- **鉴权机制：** 后端 API 需对非公开接口进行拦截，验证用户登录状态。

- **数据隔离：** SQL 查询必须严格限定 user_id，防止用户查看到他人的账单数据。

### 四、数据库设计摘要

根据业务需求，底层数据存储主要包含以下实体表：

- sys_user：存储用户鉴权信息。

- asset_account：存储资产账户信息及余额。

- category：存储收支分类定义。
- transaction_record：存储具体的流水账单，关联用户、账户与分类。

- budget：存储用户设定的预算目标。

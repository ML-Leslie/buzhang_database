# 个人记账系统 (BuZhang Database) \- 项目工作分配说明书

## 1\. 团队分工概述

本项目由两名成员共同完成，为了保证工作量的均衡以及技术成长的全面性，我们采用了按业务模块垂直切分的分工模式。每位成员均需独立负责自己模块内的数据库设计、后端 API 开发以及前端页面实现。

| 学号     | 成员角色   | 负责主要模块             | 占比|
| :------- | :--------- | :----------------------- |:-------|
| - | **-（A）** | **核心交易与资产模块**   | 50%  |
| - | **-（B）** | **用户、系统与报表模块** | 50%  |

## 2\. 详细任务分配

### 2.1 成员 A：

**主要职责**：负责系统最核心的“钱”的流转逻辑，确保数据的准确性与一致性。

#### (1) 后端开发 (Spring Boot)

* **资产账户模块 (AccountController, AssetAccount)**：  
  * 实现账户的增删改查接口。  
  * **核心难点**：处理账户余额的原子性更新。  
* **交易记录模块 (TransactionController, TransactionRecord)**：  
  * 实现记账（收入/支出）接口、删除账单接口。  
  * **核心逻辑**：实现 BookkeepingService 中的事务逻辑——当新增/删除账单时，自动同步扣减/增加对应资产账户的余额。  
* **数据库设计**：  
  * 负责 asset\_account 表和 transaction\_record 表的结构设计与 SQL 编写。

#### (2) 前端开发 (React)

* **记账交互组件**：  
  * 开发复杂的记账弹窗（包含金额输入、日期选择等）。  
* **交易列表页面**：  
  * 实现交易流水的列表展示。  
  * 实现按“年-月”筛选流水数据的功能。  
* **资产管理页面**：  
  * 实现账户卡片的展示与编辑功能。

### 2.2 成员 B：系统架构与数据分析工程师

**主要职责**：负责系统的基础架构搭建、用户安全以及数据的统计分析展示。

#### (1) 后端开发 (Spring Boot)

* **用户与安全模块**：  
  * 实现注册、登录接口。  
    
* **分类与预算模块**：  
  * 实现收支分类的自定义管理（图标、名称）。  
  * 实现预算的设定与超支预警逻辑。  
* **统计报表接口**：  
  * 编写复杂的 SQL 聚合查询（如：按月统计各分类总支出、计算每日收支趋势），为前端图表提供数据支持。

#### (2) 前端开发 (React)

* **基础架构搭建**：  
  * 负责项目脚手架搭建（Vite \+ React Router），配置 Axios 拦截器（统一处理 Token）。  
  * 开发公共布局组件 Sidebar（侧边栏导航）。  
* **用户页面**：  
  * 实现登录与注册表单的校验及交互。  
* **仪表盘与报表**：  
  * **核心难点**：集成图表库（如 Recharts/ECharts），实现饼图（收支构成）和折线图（收支趋势）的渲染。  
* **设置页面**：  
  * 实现 CategoryManagement 分类管理页面。
